# Git Setup Instructions

## Подготовка к загрузке на GitHub

Этот проект настроен для загрузки на Git с исключением больших файлов моделей и временных файлов.

### Что исключено из репозитория

`.gitignore` настроен на исключение следующих файлов:

1. **Модели и адаптеры** (большие файлы):
   - `126_csdn_disQwen0.5B/`
   - `rockyou_100w_disQwen0.5B/`
   - Все файлы `*.safetensors`, `*.bin`, `*.pt`, `*.pth`

2. **Выходные файлы**:
   - Директория `outputs/`
   - CSV файлы с результатами
   - JSON логи эволюции

3. **Временные файлы**:
   - Python cache (`__pycache__/`)
   - Виртуальные окружения
   - IDE конфигурации
   - Jupyter checkpoints

4. **Системные файлы**:
   - `.DS_Store`, `Thumbs.db`

### Инициализация Git репозитория

```bash
# Перейти в директорию проекта
cd password-guessing

# Инициализировать Git репозиторий
git init

# Добавить все файлы (исключая те, что в .gitignore)
git add .

# Создать первый коммит
git commit -m "Initial commit: Password guessing evolution project"

# Добавить удаленный репозиторий (замените URL на ваш)
git remote add origin https://github.com/your-username/password-guessing.git

# Загрузить на GitHub
git push -u origin main
```

### Важные замечания

- **Модели не включены**: Пользователи должны будут скачать или обучить модели самостоятельно
- **Данные**: Файлы `data/train.json` и `data/test.json` исключены. Добавьте инструкции по их получению в README
- **Конфигурация**: Файлы конфигурации включены в репозиторий

### Рекомендации

1. Обновите основной `README.md` с инструкциями по установке моделей
2. Добавьте скрипт для автоматической загрузки моделей (если возможно)
3. Документируйте требования к данным
4. Укажите версии зависимостей в `requirements.txt`

### Структура репозитория

После загрузки на Git, репозиторий будет содержать:
- Исходный код (`src/`)
- Конфигурационные файлы (`configs/`)
- Скрипты запуска
- Документацию
- Примеры промптов (`data/prompt.txt`)

**Не будет содержать**:
- Обученные модели
- Выходные файлы
- Большие датасеты
